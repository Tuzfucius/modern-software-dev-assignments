## 作业概述
以 3 种不同的技术栈构建相同的功能性 Web 应用。至少一个版本必须使用 [`bolt.new`](https://bolt.new/)（AI 应用生成平台）创建。至少一个版本必须使用非 JavaScript 语言用于前端或后端（例如 Django、Ruby on Rails）。

你可以重用前几周的应用（"开发者控制中心"）或创建新应用，只要它满足[最低功能范围](#最低功能范围)即可。应用应该是端到端功能性的（前端 + 后端 + 适当处的持久化）并展示连贯的功能集。

## 最低功能范围
- 用户可以创建、读取、更新和删除主要资源（例如笔记、任务、帖子）。
- 适当处的持久化存储（数据库或基于文件）。
- 基本验证和错误处理。
- 简单但功能性的 UI，显示主要流程。
- 清晰说明如何本地运行每个版本（如果部署的话也提供部署链接）。

## 技术栈要求
构建相同应用的 3 个独立版本，每个使用不同的技术栈。示例：
- MERN（MongoDB、Express、React、Node.js）
- MEVN（MongoDB、Express、Vue.js、Node.js）
- Django + React（或 Vue）
- Flask + Vanilla JS（或 React）
- Next.js + Node（或 NestJS）
- Ruby on Rails（全栈）

请注意，至少一个版本必须包括前端或后端的非 JavaScript 语言（例如 Python/Django、Ruby/Rails）。

至少一个版本必须使用 AI 应用生成平台 **[`bolt.new`](https://bolt.new/)** 构建，但你可以随意探索其他应用生成平台（例如 Lovable、Figma Make）用于其他版本。

## 了解 Bolt
Bolt 是一个 AI 辅助开发平台，可以从自然语言提示生成网站、Web 应用和移动应用。用户可以用纯文本描述他们的想法，Bolt 会在几分钟内生成一个功能原型——从落地页和电子商务网站到 CRM 和移动工具。了解更多[这里](https://support.bolt.new/building/intro-bolt)。

## AI 应用生成器使用提示
- 就像 Bolt 这样的应用生成器最适合现代全栈技术，不指定特定框架时你将默认获得这些。
- 最好从描述你的应用概念、实体、路由和 UI 流程的清晰提示开始。
- 在提示中清楚地描述数据模型和关系。
- 迭代地优化数据模型、CRUD 端点、认证（如果使用）和前端组件的提示。
- 保持每个版本隔离以避免依赖冲突。
- 导出或同步生成的代码并将其作为该技术栈的独立项目文件夹提交。
